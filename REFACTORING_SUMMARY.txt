GEOMETRY & GRID HANDLING
=========================

The refactored code correctly handles both uniform and probit grids:

  # Uniform grid (original "good run")
  binning: uniform
  → Equal-sized bins: Δu_i = Δv_j = 1/m
  → ISE = Σ (p_ij - q_ij)^2 Δu_i Δv_j (reduces to mean when uniform)

  # Probit grid (boundary-focused)
  binning: probit
  → Smaller bins near 0 and 1, larger bins in center
  → ISE = Σ (p_ij - q_ij)^2 Δu_i Δv_j (correct integration)
  → Correctly down-weights tiny boundary cells

Key functions:
  - build_binning(m, mode='uniform'|'probit') → edges, centers, widths
  - build_geometry(m, mode, device) → dict with area, row_widths, col_widths
  - compute_density_losses(...) → uses geometry['area'] for correct ISE


LOSS FUNCTION NOTES
===================

1. ISE (Integrated Squared Error)
   - Formula: ∫∫ (p-q)^2 du dv ≈ Σ (p_ij - q_ij)^2 Δu_i Δv_j
   - Correctly accounts for variable cell sizes on non-uniform grids

2. Cross-Entropy
   - Formula: -Σ (q_ij Δu_i Δv_j) log(p_ij Δu_i Δv_j)
   - Treats density grid as a discrete distribution

3. Marginal KL
   - Formula: Σ p_i log p_i (row) + Σ p_j log p_j (col)
   - Can be NEGATIVE for nearly uniform marginals (entropy interpretation)
   - This is mathematically correct, not a bug

4. Tail Loss
   - Focuses on boundary regions (τ=0.15 by default)
   - Formula: Σ mask_ij (log p_ij - log q_ij)^2
   - Boundary config uses tail_weight=2.0 (4x higher than standard 0.5)

KNOWN ISSUES & SOLUTIONS
========================

Issue: "Loss is negative"
  → Cause: Marginal KL term (Σ p log p) is negative for nearly uniform marginals
  → Solution: This is correct behavior (related to entropy). If undesired,
              set loss_weights.marg_kl: 0.0 or use true KL divergence.

Issue: "Fits are bad near boundaries despite low loss"
  → Cause: Training data clips densities at 1000, but eval uses wider range (1e6)
  → Solution: Use boundary-focused config (higher tail_weight) or match clipping
              ranges between training and evaluation.

Issue: "Probit grid has much larger initial loss"
  → Cause: Smaller boundary cells amplify density prediction errors
  → Solution: This is expected. Monitor relative improvement, not absolute values.
              The ISE integration is now correct (area-weighted).
